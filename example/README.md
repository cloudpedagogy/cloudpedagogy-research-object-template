# Missed Appointment Study: A Synthetic Example

Example Project for the CloudPedagogy Research Object Template

------------------------------------------------------------------------

## Overview

This is a fully self-contained, synthetic Quarto website demonstrating
the **CloudPedagogy Research Object Template**.

It illustrates how a completed, governance-ready research object should
look when the template is fully populated.

🔙 Return to main template repository:\
[CloudPedagogy Research Object Template](../README.md)

------------------------------------------------------------------------

## Project Description

This fictional study examines factors associated with missed outpatient
appointments.

It uses a purely synthetic dataset and:

-   Does **not** represent any real institution, clinic, geography, or
    health system\
-   Uses no real patient data\
-   Contains no identifiable information\
-   Was created solely for demonstration purposes

The goal is to demonstrate structure, governance artefacts, and optional
AI reflection --- not to model real clinical outcomes.

------------------------------------------------------------------------

## What This Example Demonstrates

This example shows how to:

-   Define a clear research question\
-   Document data sources and pre-processing steps\
-   Specify a statistical model\
-   Present results transparently\
-   Complete governance artefacts (dataset card, model card, risk
    register, decision log)\
-   Document proportionate AI tool usage (optional)\
-   Render a structured research object using Quarto

It is intended as a working reference implementation.

------------------------------------------------------------------------

## Who This Is For

This example is intended for:

-   Researchers learning to use the template\
-   Teams exploring governance-ready research documentation\
-   Healthcare analytics groups seeking structural consistency\
-   AI-in-health projects needing proportionate disclosure practices

------------------------------------------------------------------------

## Quickstart

From the root of the repository:

``` bash
cd example
quarto preview
```

To build the static site:

``` bash
quarto render
```

The rendered output will be available in the `_site/` directory.

All data, models, risks, findings, and governance documentation in this
