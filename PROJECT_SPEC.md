# PROJECT_SPEC: cloudpedagogy-research-object-template

## 1. Repo Name
`cloudpedagogy-research-object-template`

## 2. One-Sentence Purpose
A standardized template for authoring research outputs and technical documentation with embedded AI governance metadata.

## 3. Problem the App Solves
The lack of standardized transparency in AI-assisted research outputs; provides a consistent structure for tracking and declaring AI usage and governance completion.

## 4. Primary User / Audience
Researchers, postgraduate students, and technical writers within the CloudPedagogy ecosystem.

## 5. Core Role in the CloudPedagogy Ecosystem
The "Publication Layer"; used to generate the human-readable research findings, technical guides, and reports produced by the other ecosystem tools.

## 6. Main Entities / Data Structures
- **ProjectMetadata**: YAML frontmatter (tracking Title, Version, `ai_used`, `governance_complete`).
- **Research Object**: The final bundled output (HTML/PDF) representing a "Research Object" in the ecosystem.

## 7. Main User Workflows
1. **Initialize Project**: Scaffold a new documentation or research repository from this template.
2. **Configure Metadata**: Complete the `project-metadata.yml` declaration, specifically tracking AI involvement.
3. **Author Content**: Write in Markdown (.qmd) using the pre-defined SCSS/styling system.
4. **Publish**: Build and deploy the static result as a governed research object.

## 8. Current Features
- Integrated Quarto build support.
- Specialized SCSS styling (CloudPedagogy theme).
- Lightweight "Governance Ledger" via YAML-based declaration.
- Support for static HTML and PDF output formats.

## 9. Stubbed / Partial / Incomplete Features
- Not explicitly defined.

## 10. Import / Export and Storage Model
- **Storage**: File-system based (.yml, .qmd).
- **Output**: Static files generated via the build process.

## 11. Relationship to Other CloudPedagogy Apps
Acts as the final "Display" or "Communication" layer for findings generated by the mapping, simulation, and governance tools.

## 12. Potential Overlap or Duplication Risks
Overlaps with generic academic templates; distinguished by its metadata fields geared specifically towards AI-governance declarations.

## 13. Distinctive Value of This App
Treats "AI Declaration" as a first-class citizen of the research object's metadata, making it easier to track AI-involvement across a library of publications.

## 14. Recommended Future Enhancements
(Inferred) Automated population of the metadata YAML from `decision-record` or `governance-scanner` JSON exports; bibtex-aware AI attribution.

## 15. Anything Unclear or Inferred from Repo Contents
Use of Quarto as the primary engine is central to the project's build logic and file structure.
